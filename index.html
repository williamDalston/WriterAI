üöÄ Deploy WriterAI

üìå Important: WriterAI is a Python FastAPI application. Host it on a platform that supports long-running Python services (Render, Railway, Fly.io, etc.).

Option 1: Deploy to Render (Recommended)

One-click deployment with automatic HTTPS and GitHub integration.

1. Fork https://github.com/williamDalston/WriterAI
2. Visit render.com and create a Web Service
3. Point to your fork and pick the Python runtime
4. Set OPENAI_API_KEY in Environment Variables
5. Deploy and watch the logs

‚úÖ Live app: https://your-app-name.onrender.com

Option 2: Run Locally

Perfect for development and quick experiments.

# Clone the repository
git clone https://github.com/williamDalston/WriterAI.git
cd WriterAI

# Install dependencies (Poetry managed)
make install

# Configure environment
cp env.template .env
# Add OPENAI_API_KEY=... to .env

# Start the REST API
make serve

# Start the web dashboard
make serve-web
# Open http://localhost:8080

Option 3: Docker Anywhere

Package everything into a container for any Docker-ready host.

# Build image
docker build -t writerai .

# Run container
docker run -p 8080:8080 -e OPENAI_API_KEY=sk-xxx writerai

# Dashboard: http://localhost:8080
# API: http://localhost:8080/api/v2/docs

# 1. Create a project interactively (stores config under configs/<slug>.yaml)

--state data/the_last_starship/state_snapshots/latest_state.json \
--output-dir outputs/the_last_starship \

üå± Narrative Seeds & Idea Library

Jumpstart your project with the narrative seed generator or explore the curated database of 899 prompts inspired by genres, tropes, and themes‚Äîincluding the Grateful Dead anthology that ships with the repo.

# Generate a seed YAML blueprint
python cli.py generate-seed \
--prompt "A folk singer's songs ripple through time" \
--genre "fantasy" \
--output seeds/time_ripples.yaml \
--show-summary

# Browse ideas visually
make serve-web
# Visit http://localhost:8080/ideas

# Query the idea database via CLI
make db-search QUERY="parallel realities"
  • Seeds capture title, genre, synopsis, character dossiers, themes, and marketing hooks.
  • Resumable checkpoints are written to data/<slug>/state_snapshots after each stage.
  • Idea library accessible via FastAPI (GET /api/v2/ideas/search) or the dashboard search UI.

üîå Automation & API Highlights

The REST API shares the same orchestrator as the CLI, complete with API-key authentication, structured logging, and async background workers for long-running generation tasks.

# Start API (uses uvicorn under the hood)
make serve

# Generate API key (edit .env or env_config)
echo "WRITERAI_API_KEY=local-demo-key" >> .env

# Create a project through the API
curl -X POST http://localhost:8000/api/v2/projects \
-H "X-API-Key: local-demo-key" \
-H "Content-Type: application/json" \
-d '{
"title": "Terrapin Crossroads",
"genre": "fantasy",
"synopsis": "A wanderer arrives at a cosmic crossroads..."
}'
